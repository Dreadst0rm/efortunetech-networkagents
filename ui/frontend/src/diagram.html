0 CLI Entry & Config Load
flag.Parse() — Parse -config, -daemon, -feed, -output
config.Load() — Merge JSON with defaults, validate thresholds & whitelist
⤉
1 System Discovery
systeminfo.Gather()
os.Hostname() + runtime.GOOS + net.Interfaces()
Returns: SystemDetails{hostname, OS, localIPs, MACs}
⤉
2 Network Scan
scanner.ScanAll(cfg)
enumerateProcesses() — platform-specific (wmic /proc /lsof)
getNetConnections() — platform-specific (netstat /proc/net /lsof)
Correlate PID → process name
determineDirection() — outbound / internal / inbound
Filter excluded PIDs/processes from config
processinfo.GetProcessInfo(pid) for each unique PID
⤉
DNS Resolution & Capture
dns.ResolveConnectionsDNS(conns, concurrency)
Parallel worker pool (default 10 concurrent)
2s timeout per lookup, deduplicates addresses
Populates c.DNSName for resolved connections
dns.CaptureDNSQueries() — platform-specific DNS cache capture
Cross-reference: ipToDomain map → populate remaining DNSNames
⤉
3 Baseline Comparison
baseline.Load(baseline.json)
baseline.Diff(current, previous) — New / Gone / Unchanged
baseline.Save(current) — overwrite with current state
⤉
4 Threat Intel Aggregation
threatintel.NewThreatIntelDB()
Built-in: KnownC2IPs (33 indicators)
External file: -feed flag → GetFeedIOCs()
Live: ThreatFox API → FeedCacheManager (1h TTL)
C2IntelFeeds CSV: FetchAllIOCs() (~590 indicators)
⤉
4 Risk Assessment
scanner.AssessConnectionRiskWithThreatIntel()
6 heuristics per connection:
1. Suspicious port (4444, 8080, 1337...)
2. Suspicious process (cmd.exe, powershell...)
3. Transition state (SYN_SENT, TIME_WAIT...)
4. High IP connection count (>= threshold)
5. High process connection count (>= threshold)
6. Privilege escalation chain (elevated + unsigned + temp path)
Threat intel boost: confidence >= 80 → high
⤉
5 Report Generation & Alerts
report.GenerateMarkdown() / GenerateJSON() / GenerateCSV()
Markdown: system info, connections, risks, privilege escalation, baseline
JSON: full structured data with findings summary
CSV: connections + risks spread sheets
alerting.Registry.Send() — Webhook + Stdout for Critical/High
⤉
Ø Output Artifacts
network_sentinel__.md
network_sentinel__.json
_connections.csv / _risks.csv
baseline.json (saved current state)
captured_dns_queries__.json

Data Types

Core Models

  • Connection — PID, process, local/remote addr:port, protocol, state, direction, DNSName
  • ConnectionRisk — Connection + RiskLevel, RiskReasons[], IsSuspicious, IsWhitelisted
  • RiskLevel — "low" | "medium" | "high" | "critical"
  • ProcessEntry — PID, Name

Security

  • processinfo.Info — PID, name, username, exePath, privLevel, isSigned, integrity
  • AdminPrivilegeLevel — "elevated" | "standard" | "system"
  • IntegrityLevel — System / High / Medium / Low

Intelligence

  • IOC — indicator, type, malwareFamily, confidence, country, tags, source
  • ThreatIntelDB — ipv4 map, domain map

Reports

  • report.Data — System, Connections, Processes, Risks, Security, Baseline, Whitelist, DNSQueries
  • Findings — outbound, endpoints, risk counts, privEsc, whitelisted

Config

  • Config — Thresholds, Excluded, Whitelist, DNS, Alerting, ThreatIntel
  • Thresholds — minIP, minProcess, criticalThreshold, highThreshold
